Dieter Rams' 10 Principles Audit

1. Good Design is Innovative
7/10
Evidence: Subtle innovations like the footer easter egg animation, tag-based recommendation algorithm, and blue-tinted actionable content boxes. However, the overall experience follows conventional patterns.
Concrete Fix:

Add micro-interactions to filter tags: When clicking a tag, animate the list items fading in/out with staggered timing (50ms delay between each). This creates a sense of curated intelligence without being flashy.

transition: opacity 300ms ease; transition-delay: calc(var(--item-index) * 50ms);
2. Good Design Makes a Product Useful
9/10
Evidence: Excellent information architecture - "About" (credibility), "Now" (current work), "Projects" (portfolio), "Writing" (ideas). Clear navigation, scannable lists, actionable CTAs on Now page.
Concrete Fix:

Add keyboard navigation: Let users press numbers 1-4 to navigate to main sections, "/" to focus search (if added later). Shows thoughtfulness for power users.

document.addEventListener('keydown', (e) => { if (e.key === '1') window.location = '/about'; });
3. Good Design is Aesthetic
8/10
Evidence: Beautiful dark minimalist aesthetic with cream text on deep charcoal. However, some inconsistencies exist - H4 headings on Now page use colons, breadcrumb on Topics page uses "/" separator inconsistently.
Concrete Fix:

Unify breadcrumb style: Topics page shows "Max Milne / Writing" - change to use bullet separator like filters: "Max Milne • Writing" for visual consistency with the rest of the site.

4. Good Design Makes a Product Understandable
7/10
Evidence: Generally clear, but some areas lack hierarchy. On Now page, H4 headings like "Starting Talk. Art. Therapy.:" compete visually with body text. The site navigation shows all items at same weight.

Concrete Fix:

Remove colons from H4 headings and increase weight: Change "Starting Talk. Art. Therapy.:" to "Starting Talk. Art. Therapy." and bump font-weight from 400 to 600. Makes section breaks clearer.

5. Good Design is Unobtrusive
9/10
Evidence: Excellent restraint - no popups, no cookie banners, no tracking notices, no newsletter prompts. The footer animation is subtle and doesn't demand attention. Content is the hero.
Concrete Fix:

Reduce external link indicators: The "↗" symbol on external links is slightly obtrusive. Consider making it appear only on hover, or use a more subtle indicator like slight underline color change.

.external-link::after { opacity: 0; transition: opacity 200ms; } .external-link:hover::after { opacity: 1; }
6. Good Design is Honest
10/10
Evidence: Perfect authenticity - the voice is genuine ("I enjoy function and public utility"), no inflated claims, "Coming soon" states clearly when content isn't ready. The tagline "Therapist, Artist, Contemplative" is direct and true.
Concrete Fix:

None needed. This is the site's strongest principle. The honesty in copy and design creates trust immediately.

7. Good Design is Long-lasting
9/10
Evidence: Timeless aesthetic choices - dark minimalism, systematic spacing, classic typography. No trendy gradients, glassmorphism, or effects that will date quickly. The design will age well.
Concrete Fix:

Document the design system: Create a simple style guide showing the spacing scale, color palette, and typography choices. This ensures future additions maintain the timeless quality.

Note: This has already been partially done in STYLE_GUIDE.md - continue to maintain it.

8. Good Design is Thorough Down to the Last Detail
6/10
Evidence: Some details need attention - breadcrumb inconsistency on Topics page, H4 colons on Now page, "Linked mentions" section shows "There are no notes linking to this note" (negative affordance), filter capitalization inconsistent between pages.
Concrete Fix:

Hide empty "Linked mentions" section: Instead of showing "There are no notes linking to this note," simply don't render the section if there are no backlinks. Negative space is better than negative information.

{% if backlinks.size > 0 %} [show section] {% endif %}
9. Good Design is Environmentally Friendly
8/10
Evidence: Dark theme reduces energy consumption on OLED screens. Static site generation = minimal server resources. No tracking scripts or heavy JavaScript frameworks. Could optimize image assets further.
Concrete Fix:

Add WebP image format with fallbacks: Convert images to WebP format (typically 30% smaller than PNG/JPEG) with fallback to original format for older browsers.

<picture><source srcset="image.webp" type="image/webp"><img src="image.png"></picture>
10. Good Design is as Little Design as Possible
9/10
Evidence: Excellent minimalism - no unnecessary decorations, clear typography, generous whitespace. Every element serves a purpose. The only slight excess is the separation lines on some pages.

Dieter Rams Weighted Summary

Overall Score: 8.2/10

The site demonstrates strong adherence to Rams' principles, particularly excelling in honesty (10/10), usefulness (9/10), and restraint (9/10).

The main opportunities lie in thoroughness (small inconsistencies in details) and understandability (some hierarchy improvements needed).

This is fundamentally excellent work that respects the user's attention and intelligence. The suggested fixes are refinements, not major corrections.

Calm Technology Cross-Check

Evaluating elements that demand unnecessary attention (motion, contrast, density). Goal: maintain personality while lowering cognitive load.

Motion
Excellent

Finding: Only one animated element - the footer dash animation. It's subtle, pauseable on hover, and doesn't distract.

Recommendation: Perfect as is. Consider adding `prefers-reduced-motion` support to disable it entirely for users who request it.

Contrast
Needs Attention

Finding: The blue accent color (#93c5fd) on dark background is high contrast. While accessible, it demands attention even when not interactive.

Recommendation: Reserve the bright blue only for active states and primary CTAs. Use --color-warm-gray for secondary elements like "You might also enjoy."

Density
Excellent

Finding: Great use of whitespace throughout. Lists are scannable, text blocks are comfortably wide (max 42rem), generous margins.

Recommendation: No changes needed. This is exemplary calm density.

Notifications
Excellent

Finding: Zero notifications, popups, or interruptions. No cookie banners, newsletter prompts, or chat widgets.

Recommendation: Perfect. Never add these.

Navigation Permanence
Excellent

Finding: Navigation stays in place, no sticky headers, no disappearing menus. Users always know where they are.

Recommendation: Consider adding subtle active state to current nav item (slightly brighter or underlined).

Cognitive Load
Excellent

Finding: Each page has one clear purpose. Information hierarchy is clean. Tag-based recommendations reduce decision fatigue.

Recommendation: Consider adding page descriptions under H1 titles to set context immediately (like Homepage does).

Calm Technology Summary

The site scores exceptionally well on Calm Technology principles. It respects the visitor's attention and never demands unnecessary focus.

Key Strength: Complete absence of dark patterns - no popups, tracking notices, or interruptions.

One Refinement: The bright blue accent color could be reserved for truly actionable elements only, using softer grays for passive content like recommendations.

Innovation (7.0/10)

5 Micro-Interactions / Details for Innovation

These additions would register as "innovation" without breaking minimalism:

  1. Staggered filter animations: When clicking a tag filter, fade out non-matching items with 50ms delay between each, creating a "dealt cards" effect
  2. Reading progress indicator: On long note pages, show a subtle progress line at top (1px height, blue accent color) that grows as you scroll
  3. Hover-reveal publication dates: On Projects list, show year by default, but reveal full date on hover of the year column (smooth opacity transition)
  4. Contextual navigation hints: When hovering over a writing title, show the tags associated with it as a subtle fade-in beneath
  5. Smart link previews: Internal links could show a 2-line excerpt preview on hover after 500ms delay (like Wikipedia), helping users decide if they want to navigate

Top 5 Quick Wins (Least Effort / Largest Impact)

Prioritized by implementation time vs. visible improvement. Each takes <15 minutes but meaningfully improves the experience.

  1. Hide Empty "Linked Mentions" Section
    Effort: 2 minutes (add conditional to template)
    Impact: Removes negative affordance, cleaner page endings
    Code: {% if backlinks.size > 0 %} [section] {% endif %}
  2. Unify Breadcrumb Style (• instead of /)
    Effort: 1 minute (change one character)
    Impact: Visual consistency across entire site
    Code: Change Max Milne / Writing to Max Milne • Writing
  3. Increase H4 Font-Weight & Remove Colons
    Effort: 5 minutes (CSS + content update)
    Impact: Much clearer section hierarchy on Now page
    Code: h4 { font-weight: 600; } and remove ":" from headings
  4. Add Active State to Navigation
    Effort: 10 minutes (Liquid logic + CSS)
    Impact: Users always know where they are
    Code: {% if page.url contains '/about' %}class="active"{% endif %}
  5. Add prefers-reduced-motion Support
    Effort: 3 minutes (one media query)
    Impact: Respects user preferences, demonstrates care
    Code: @media (prefers-reduced-motion: reduce) { * { animation: none !important; } }

Implementation Priority

These five changes would take approximately 20 minutes total but would polish the site significantly. They address the most visible inconsistencies and add thoughtful touches that separate good work from great work.

After these quick wins, consider the micro-interactions from the Innovation section for a second phase of refinement.
